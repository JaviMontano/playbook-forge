# KATA-SPEC v1.0 -- 5-Level Shu-Ha-Ri Progression System

> **Entrusted Standard** for the 5-level kata system that drives habit
> formation in every playbook generated by `playbook-forge`. Defines level
> definitions, practice structures, advancement criteria, flow activation
> matrices, and learning traps.
>
> Derived from golden reference: `playbook_bu2_agile_pm_excellence_v3.html`

## 1. The 5 Kata Levels

### KA-1: Observar / Observe

| Field | Value |
|---|---|
| Level | KA-1 |
| Name ES | Observar |
| Name EN | Observe |
| Shu-Ha-Ri Phase | Shu (守) -- Follow the form |
| Duration | 15 min |
| Color | `var(--sofka-orange)` |
| Color class | orange (Shu indicator) |

**Concept**: Before attempting any flow, observe it completely. Read all
steps without executing. Build a mental model of input, output, and purpose.
This prevents "cargo-culting" -- following steps without understanding why.

**Practice Structure (4 trials)**:

| Trial | Name ES | Name EN | What happens |
|---|---|---|---|
| Trial 1 | Lee el flujo completo | Read the complete flow | Read all 4 steps of a flow modal without executing anything. Identify: what goes in, what comes out, how long it takes. |
| Trial 2 | Imagina el output | Imagine the output | Without opening any tool, write down what the output should look like. Build expectation before seeing the real result. |
| Trial 3 | Identifica el "por que" | Identify the "why" | Ask: what problem does this flow solve? Write 1 concrete situation from your work where this flow would have changed the outcome. |
| Trial 4 | Siguiente paso | Next step | Confirm readiness for KA-2. The difference between an informed imitator and a blind imitator is this 15-minute exercise. |

**Advancement Criterion**: You can describe the expected output of a flow
before executing it. You can name the specific problem it solves in your work.
If you can do both -- you move to KA-2.

---

### KA-2: Imitar / Imitate

| Field | Value |
|---|---|
| Level | KA-2 |
| Name ES | Imitar |
| Name EN | Imitate |
| Shu-Ha-Ri Phase | Shu (守) -- Follow the form |
| Duration | 30 min |
| Color | `var(--sofka-orange)` |
| Color class | orange (Shu indicator) |

**Concept**: Execute the flow exactly as documented. No modifications,
no optimizations -- just follow the form. The 4 repetitions are not to
produce perfect outputs -- they are to build muscle memory. Optimization
comes in KA-3.

**Practice Structure (4 trials)**:

| Trial | Name ES | Name EN | What happens |
|---|---|---|---|
| Trial 1 | Primera vez | First time | Check the modal at each step. Normal to take twice as long. Goal: complete the full cycle without skipping any step. Output probably needs correction. |
| Trial 2 | Patron | Pattern | Check the modal only when needed. Output should be usable with minor adjustments. Identify: which part consumes most time? |
| Trial 3 | Fluencia | Fluency | Without checking the modal. If stuck, try to reconstruct from the flow's logic before looking. Output should be publish-ready. |
| Trial 4 | Criterio | Criterion | Completed in under 30 min without checking the modal. Output published same day. Measure: did the output change how the team works? |

**Advancement Criterion**: Trial 4 completed in under 30 min without
consulting the modal. Output published same day. The complete flow cycle
is in your memory without needing external reference. Achieved 4 times
consecutively.

---

### KA-3: Adaptar / Adapt

| Field | Value |
|---|---|
| Level | KA-3 |
| Name ES | Adaptar |
| Name EN | Adapt |
| Shu-Ha-Ri Phase | Ha (破) -- Break the form |
| Duration | 30 min |
| Color | `var(--sofka-info)` |
| Color class | info blue (Ha indicator) |

**Concept**: Modify flows consciously to fit your specific context without
losing the expected output. You change the packaging, not the content.
If your version's output is worse than the original -- it is not adaptation,
it is degradation.

**Practice Structure (4 trials / adaptation types)**:

| Trial | Name ES | Name EN | What happens |
|---|---|---|---|
| Trial 1 | Contexto de proyecto | Project context | Add project-specific context to the base prompt. More precise output, same time. |
| Trial 2 | Naming y ubicacion | Naming and location | Adjust naming conventions and publication destinations to your organization. Structure stays; destination adapts. |
| Trial 3 | Formato de output | Output format | Modify templates to match organizational standards. The 4 components of a Quad-Doc stay; the visual format may change. |
| Trial 4 | Prompt iterativo | Iterative prompt | Improve the prompt itself based on patterns from previous executions. Each iteration should produce better output without manual editing. |

**Advancement Criterion**: Your adapted version produces better output
than the standard in 3 consecutive executions. "Better" = faster, more
accurate, or more relevant to your context -- while maintaining all required
output components.

---

### KA-4: Ensenar / Teach

| Field | Value |
|---|---|
| Level | KA-4 |
| Name ES | Ensenar |
| Name EN | Teach |
| Shu-Ha-Ri Phase | Ri (離) -- Transcend the form |
| Duration | 30 min |
| Color | `var(--sofka-positive-text)` |
| Color class | positive gold (Ri indicator) |

**Concept**: Teaching reveals your own gaps. Where the learner gets
confused = where you still have uncertainty. F1-F12 from memory. Teach
a colleague. If they can execute F1-F4 alone without re-explanation,
you have truly internalized the system.

**Practice Structure (4 trials)**:

| Trial | Name ES | Name EN | What happens |
|---|---|---|---|
| Trial 1 | Explicar sin herramienta | Explain without tool | Describe a flow to a colleague without opening any tool. If you cannot explain it clearly, you do not understand it well enough. |
| Trial 2 | Guiar en vivo | Guide live | Walk a colleague through a flow execution in real time. Observe where they hesitate -- those are your teaching (and understanding) gaps. |
| Trial 3 | Evaluar output | Evaluate output | Review a colleague's flow output. Can you identify what is missing or wrong without checking the modal? |
| Trial 4 | Soltar | Release | The colleague executes F1-F4 alone. If they need to ask you again, the teaching (and your mastery) is not complete yet. |

**Advancement Criterion**: A colleague executes F1-F4 alone without
re-explanation. They produce publish-ready output independently.

---

### KA-5: Crear / Create

| Field | Value |
|---|---|
| Level | KA-5 |
| Name ES | Crear |
| Name EN | Create |
| Shu-Ha-Ri Phase | Ri (離) -- Transcend the form |
| Duration | 60 min |
| Color | `var(--sofka-positive-text)` |
| Color class | positive gold (Ri indicator) |

**Concept**: Design Flow 13 -- your own AI-native flow. This means creating
all 4 artifacts: prompt, template, NotebookLM notebook configuration, and
Gem configuration. Then publish it and have someone else execute it without
your help.

**Practice Structure (4 trials)**:

| Trial | Name ES | Name EN | What happens |
|---|---|---|---|
| Trial 1 | Identificar proceso | Identify process | Find a repetitive process in your work that is not covered by F1-F12. Define: input, output, frequency, tool candidates. |
| Trial 2 | Disenar artefactos | Design artifacts | Create the 4 artifacts: prompt template, output template, NLM notebook config, Gem system prompt. |
| Trial 3 | Probar internamente | Test internally | Execute your custom flow 4 times. Each execution should take less time. Output should be consistent. |
| Trial 4 | Publicar y validar | Publish and validate | Another person executes your flow without any prior explanation from you. If they succeed, the flow is production-ready. |

**Advancement Criterion**: Your custom flow is used by another PM without
friction and without prior explanation. The 4 artifacts are complete and
the flow is documented in playbook format.

## 2. Kata x Flow Activation Matrix

This matrix defines which flows are active at each kata level. Users
MUST NOT attempt flows beyond their current kata level's activation set.

| KA Level | Active Flows | What You Practice | Advancement Signal |
|---|---|---|---|
| KA-1 | F1-F4 (read-only) | Read flow cards. Observe real outputs. Identify input-output without executing. | Describe any flow in 2 min without the card |
| KA-2 | F1-F4 (execution) | Execute F1-F4 copying steps exactly with real evidence. Zero customizations. | DoD met in <30 min without rework, 4 times consecutively |
| KA-3 | F1-F9 | F1-F4 without guide + incorporate F5-F9. Adjust prompt/format iteratively, one parameter at a time. | Your version produces better output than standard in 3 consecutive executions |
| KA-4 | F1-F12 | F1-F12 from memory. Teach a colleague. Where they get confused = gap you still have. | Colleague executes F1-F4 alone without re-explanation |
| KA-5 | F13 + custom | Design Flow 13: prompt + template + NLM notebook + Gem. Publish and have someone else execute it. | Your custom flow is used by another PM without friction |

### Activation Rules

1. **No skipping levels** -- KA-1 must be completed before KA-2
2. **No premature flow access** -- a KA-2 user should not attempt F5-F9
3. **Regression is possible** -- 3 weeks without practice triggers regression (see section 4)
4. **The matrix is a RECOMMENDATION** -- urgent needs may override, but the user acknowledges they are skipping ahead

## 3. Transition Rules

### Advancement (Level Up)

| Condition | Action |
|---|---|
| 4 consecutive successes meeting the level's criterion | Advance to next level |
| All flows in the level's activation set mastered | Advance to next level |
| Evidence of advancement published (output shared) | Advance to next level |

### Regression (Level Down)

| Condition | Action |
|---|---|
| 3 weeks without executing any flow | Regress one level |
| 3 consecutive failures at current level | Regress to previous level |
| Output quality drops below DoD threshold | Hold at current level until recovered |

### Hold (Stay at Current Level)

| Condition | Action |
|---|---|
| Mixed results (some pass, some fail) | Continue practicing at current level |
| New flows added at current level | Practice new flows before advancing |
| Context change (new project, new team) | Re-execute KA-2 trials for F1-F4 in new context |

## 4. Learning Traps per Level

Each level has a predictable error. Knowing it in advance saves weeks of
frustration. These traps MUST appear in the katas section (S4) of every playbook.

### KA-1 Trap: Skipping Observation

**What happens**: User jumps directly to KA-2 (imitate) without observing.
Result: "cargo-culting" -- following steps without understanding why.
Outputs are generic and inconsistent.

**How to avoid**: Require 15 minutes of observation before first execution.
The user must write down expected output before seeing the real one.

### KA-2 Trap: Premature Optimization

**What happens**: User starts modifying the flow before completing 4 exact
repetitions. "Step 2 could be done differently" -- NO. Shu means following
the form exactly. The 4 repetitions build muscle memory, not perfect outputs.

**How to avoid**: Strict rule: zero modifications during KA-2. If the user
notices something to improve, write it down for KA-3 -- do not change
anything yet.

### KA-3 Trap: Changing Everything at Once

**What happens**: User modifies prompt, format, destination, and naming
all at once. When the output degrades, they cannot identify which change
caused the problem.

**How to avoid**: One parameter at a time. Make one change, execute,
compare output to previous. If better, keep. If worse, revert. Then
try the next change.

### KA-4 Trap: Assuming Understanding

**What happens**: User teaches the process but cannot explain WHY each
step exists. The learner follows instructions but does not internalize.
Result: the learner cannot adapt when context changes.

**How to avoid**: For each step, the teacher must answer "why does this
step exist?" and "what breaks if you skip it?" If they cannot answer,
they are not ready to teach.

### KA-5 Trap: Over-engineering

**What happens**: User creates a flow that is too complex for anyone
else to execute. 8 steps, 3 tools, conditional branches. Result: the
flow works for the creator but nobody else adopts it.

**How to avoid**: Maximum 4 steps. Maximum 2 tools. The test: someone
who has never seen the flow can execute it in under 30 minutes on first
attempt using only the documentation.

## 6. Modal Internal Structure per KA Level

All 5 kata modals follow MODAL-SPEC Category 4 (Kata Modal). Specific
variations by level:

| KA | Concept callout question ES | Concept callout question EN | Timeline trials |
|---|---|---|---|
| KA-1 | Por que observar antes de hacer | Why observe before doing | 3 observation steps + 1 transition |
| KA-2 | La regla que todos violan en KA-2 | The rule everyone breaks in KA-2 | 4 standard trials (first/pattern/fluency/criterion) |
| KA-3 | Que significa "adaptar" | What "adapt" means | 4 adaptation types (context/naming/format/prompt) |
| KA-4 | Que significa ensenar | What teaching means | 4 teaching trials (explain/guide/evaluate/release) |
| KA-5 | Que significa crear | What creating means | 4 creation trials (identify/design/test/publish) |

## 7. Kata Requirements for New Playbooks

When generating katas for a new topic (not Agile PM):

1. **Always 5 levels** -- the Shu-Ha-Ri structure is universal
2. **Level names may be adapted** but must follow the observe-imitate-adapt-teach-create progression
3. **Duration may vary** but must follow: KA-1 shortest, KA-5 longest
4. **Trial structure always 4 trials** per level (except KA-1 which may have 3+1)
5. **Advancement criteria must be measurable** -- "feels comfortable" is NOT a valid criterion
6. **Learning traps must be included** -- one per level, specific to the playbook topic
7. **Color coding is FIXED** -- Shu=orange, Ha=blue, Ri=gold regardless of topic
