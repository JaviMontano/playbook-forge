# CONTENT-SPEC v1.0 -- Minimum Content Density per Section

> **Entrusted Standard** for content density in every playbook generated by
> `playbook-forge`. Defines the minimum number of components, required text
> fields, and sub-component requirements for each of the 12 mandatory sections.
>
> Derived from golden reference: `playbook_bu2_agile_pm_excellence_v3.html`
> A section that fails its density check is **BLOCKED** from delivery.

---

## General Rules

1. Every section MUST have a `.section-header` with:
   - `h2` bilingual (ES/EN), with `<span>` orange highlight on key word(s)
   - `p` bilingual description, minimum 20 words each language
2. Every section MUST end with `<hr class="section-divider">` (except cierre)
3. Sections marked MANDATORY cannot be omitted under any circumstances
4. Sections marked OPTIONAL may be omitted only when the playbook topic does
   not warrant them -- the brief.json must explicitly flag them as excluded

## S1: Glosario (MANDATORY)

| Component | Min | Target | Notes |
|---|---|---|---|
| `problem-card tip-card clickable-card` | 10 | 16 | Each term as a clickable card |
| `problem-card tip-card` (static) | 1 | 1 | "Otros" catch-all card, not clickable |
| `vraid-box` (architecture) | 1 | 1 | Architecture diagram (e.g. 3-layer) |
| `vraid-letter clickable-card` | 3 | 3 | One per architecture layer |
| `gem-bar` | 1 | 1 | Direct access links to tools |
| `gem-link` | 3 | 5 | Links inside the gem-bar |
| `callout callout-warning` | 1 | 1 | Prerequisite callout |
| Glossary modals (`modal-fg-*`) | 10 | 16 | One per clickable term card |

### Required per term card

- `style="border-left-color:..."` -- unique accent color
- `h4` with matching color -- bilingual term name
- `p.es` + `p.en` -- minimum 30 words each, includes "Si lo ignoras" consequence
- `role="button"` + `tabindex="0"` + `onclick="openModal('g-{slug}')"`

### Required sub-sections

- **Key terms** (`h3` bilingual): grid of term cards
- **Architecture** (`h3` bilingual): `vraid-box` with clickable layer cards
- **Direct access** (`gem-bar`): tool links
- **Prerequisite** (`callout`): what the user needs before starting

## S2: Profile (MANDATORY)

> Golden reference ID: `agile-pm`. ID varies by playbook context.

| Component | Min | Target | Notes |
|---|---|---|---|
| `apm-equation` | 1 | 1 | Value proposition equation (3 columns + 2 operators) |
| `apm-col` | 3 | 3 | Columns in the equation |
| `compare-grid` | 1 | 1 | Before/After comparison |
| `compare-col.before` | 1 | 1 | "Sin este playbook" / "Without this playbook" |
| `compare-col.after` | 1 | 1 | "Con este playbook" / "With this playbook" |
| `callout` | 1 | 2 | Context or framework callouts |
| List items in compare cols | 5 | 7 | Per column, bilingual |

### Required sub-sections

- **Equation block** (`apm-equation`): 3 competency areas connected by operators
- **Before/After** (`compare-grid`): concrete behavioral comparison, min 5 items per column
- **Framework callout** (`callout`): connects to formal methodology (DAD, PMI, etc.)

## S3: Flujos / Flows (MANDATORY)

> Golden reference ID: `flujos`

| Component | Min | Target | Notes |
|---|---|---|---|
| `map-table` | 1 | 1 | Master flow table with ALL flows |
| `map-table` rows (clickable) | 13 | 13 | One per flow, all with `onclick` |
| Table columns | 6 | 6 | #, Flow name, Tool, When, Output, KA Phase |
| `role="button"` on rows | 13 | 13 | Every row clickable |

### Required per flow row

- `td:first-child` -- flow number with accent color and `font-weight:800`
- `td:nth-child(2)` -- bilingual flow name (ES/EN spans)
- `td.jarvis-name` -- tool name(s)
- `td` bilingual -- when to use
- `td` -- expected output (bilingual if applicable)
- `td` -- KA level (KA-1 through KA-5)
- `onclick="openModal('{N}')"` + `role="button"` + `tabindex="0"`

### Flow grouping visual cues

- F1-F4 (nucleo): default row background
- F5-F6 (investigacion): `background:var(--sofka-orange-dim)`
- F7-F10 (infra): `background:var(--sofka-info-dim)`
- F11-F13 (avanzado): `background:var(--sofka-positive-dim)`

## S4: Katas (MANDATORY)

> Golden reference ID: `katas`

| Component | Min | Target | Notes |
|---|---|---|---|
| `map-table` (kata levels) | 1 | 1 | 5 rows, one per KA level |
| `map-table` (kata x flow matrix) | 1 | 1 | 5 rows mapping KA to active flows |
| Kata trap cards (`callout`) | 4 | 5 | One per level (can combine KA-1/KA-2) |
| DoD `decision-table` | 1 | 1 | Definition of Done per flow |
| DoD rows | 10 | 13 | One per flow with artifact + criterion |
| Kata modals (`modal-fka{1-5}`) | 5 | 5 | One per KA level |
| `acceptance-list` | 0 | 1 | Optional acceptance criteria list |

### Required sub-sections (in order)

1. **Kata levels table** (`map-table`): KA, Name ES/EN, Phase, Duration
2. **Kata x Flow map** (`h3` + `map-table`): KA, Active flows, What you practice, Advancement signal
3. **Traps per level** (`h3` + grid of callouts): predictable error at each level
4. **Definition of Done** (`h3` + `decision-table`): Flow, Artifact, DoD criterion, Duration

### Required per kata row

- KA identifier with color coding (KA-1/2: orange, KA-3: info, KA-4/5: positive)
- Bilingual name
- Shu-Ha-Ri phase
- Duration
- `onclick="openModal('ka{N}')"` + `role="button"` + `tabindex="0"`

## S5: Nucleo / Core (MANDATORY)

> Golden reference ID: `nucleo`

| Component | Min | Target | Notes |
|---|---|---|---|
| `jarvis-card clickable-card` | 4 | 4 | One per core flow (F1-F4) |
| `cycle-visual` | 0 | 1 | Optional visual cycle showing flow sequence |
| `cycle-step` | 0 | 4 | Steps in the cycle visual |

### Required per jarvis-card

- `.jarvis-card-header` with `.jarvis-icon.{tool-class}` + flow number
- `h3` bilingual flow name + `small` with tool + duration
- `.jarvis-card-body` with:
  - `p` bilingual description (min 20 words each)
  - 3-5 `.jf` blocks (label + value pairs): Herramienta, Input, Output, Prompt base, Anti-uso
  - `.jf-prompt` block with example prompt (bilingual)
  - `.jf-antiuse` block with anti-use warning (bilingual)
- `onclick="openModal('{N}')"` + `role="button"` + `tabindex="0"`

## S10: Ritmo / Rhythm (MANDATORY)

> Golden reference ID: `ritmo`

| Component | Min | Target | Notes |
|---|---|---|---|
| `vraid-box` | 1 | 1 | Framework box (VR-AID or equivalent) |
| `vraid-letter` | 4 | 5 | One per framework component |
| `metric-card clickable-card` | 4 | 4 | KPI cards with modal links |
| `timeline` (adoption) | 1 | 1 | 4-week adoption roadmap |
| `timeline-item` (adoption) | 4 | 4 | Weekly milestones |
| `acceptance-list` | 1 | 1 | Acceptance criteria for rhythm mastery |
| `semaforo-grid` or `semaforo-card` | 0 | 4 | Traffic light explanation cards |
| Decision-matrix modals (`modal-fdm{1-7}`) | 3 | 7 | Tool usage decision modals |
| Metric-detail modals (`modal-fm-*`) | 4 | 4 | One per metric KPI |
| Manager-profile modals (`modal-fm-novato/practicante/autonomo`) | 0 | 3 | Self-assessment modals |
| `jarvis-card clickable-card` (impact) | 0 | 1 | Impact summary card |

### Required sub-sections (in order)

1. **Framework** (`vraid-box`): framework explained with component cards
2. **Metrics** (`metrics-row`): 4 KPI cards, each clickable to metric modal
3. **Adoption timeline** (`timeline`): week-by-week progression plan
4. **Acceptance criteria** (`acceptance-list`): measurable criteria
5. **Tool decision matrix** (optional `decision-table`): when to use which tool

## S11: Empezar / Get Started (MANDATORY)

> Golden reference ID: `empezar`

| Component | Min | Target | Notes |
|---|---|---|---|
| `problem-card clickable-card` | 4 | 5 | Empathy cards linking to flow modals |
| `problem-card` (static tip) | 1 | 1 | Static recommendation card |
| `problem-grid` | 1 | 1 | Grid container |
| Role-variant modals (`modal-frv-*`) | 0 | 3 | Optional role-based modals |

### Required per empathy card

- `style="border-left-color:..."` -- accent color
- `h4` bilingual -- problem/scenario title
- `p.es` + `p.en` -- min 15 words each, describes the scenario
- `onclick="openModal('{flow-or-variant}')"` + `role="button"` + `tabindex="0"`

### Required structure

- Empathy-driven: cards describe SITUATIONS the user faces, not features
- Each card links to the flow that solves that situation
- At least 1 static tip card with general advice (no modal)

## S12: Cierre / Close (MANDATORY)

> Golden reference ID: `cierre`

| Component | Min | Target | Notes |
|---|---|---|---|
| `decision-table` (metadata) | 1 | 1 | Playbook metadata table |
| Metadata rows | 5 | 7 | Author, audience, team, version, pilot, etc. |
| `callout` (acknowledgments) | 1 | 2 | Attribution and acknowledgments |
| `gem-bar` | 0 | 1 | Optional final tool links |
| `callout` (legal/disclaimer) | 0 | 1 | Optional compliance statement |

### Required metadata fields

| Field | Required | Example |
|---|---|---|
| Author | YES | Name + role |
| Version | YES | v1, v2, v3 |
| Audience | YES | Target role/team |
| Team / BU | YES | Business unit or team name |
| Pilot | YES | Pilot project or validation source |
| Date | YES | Creation/update date |
| Framework | NO | DAD, PMI, Scrum, etc. |
