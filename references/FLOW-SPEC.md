# FLOW-SPEC v1.0 -- 13 AI-Native Flows Definition

> **Entrusted Standard** for the 13 AI-native flows that form the operational
> backbone of every playbook generated by `playbook-forge`. Defines mandatory
> fields, grouping rules, section placement, and modal patterns for each flow.
>
> Derived from golden reference: `playbook_bu2_agile_pm_excellence_v3.html`

---

## 1. Mandatory Fields per Flow

Every flow in a playbook MUST have ALL of the following fields defined.
These fields appear in the master flow table (S3), jarvis-cards (S5-S8),
and flow modals.

| Field | Key | Type | Required | Example |
|---|---|---|---|---|
| Flow number | `number` | int 1-13 | YES | `1` |
| Name ES | `nameEs` | string | YES | `Preparar sesiones efectivas` |
| Name EN | `nameEn` | string | YES | `Prepare effective sessions` |
| Tool(s) | `tools` | string | YES | `LaForja + LaReu` |
| When to use ES | `whenEs` | string | YES | `Antes de cada reunion clave` |
| When to use EN | `whenEn` | string | YES | `Before each key meeting` |
| Expected output ES | `outputEs` | string | YES | `Agenda + moderacion` |
| Expected output EN | `outputEn` | string | YES | `Agenda + moderation` |
| KA level | `kaLevel` | string | YES | `KA-2` |
| Duration | `duration` | string | YES | `30 min` |
| Frequency ES | `frequencyEs` | string | YES | `Antes de cada reunion` |
| Frequency EN | `frequencyEn` | string | YES | `Before each meeting` |
| Section group | `group` | enum | YES | `nucleo\|investigacion\|infra\|avanzado` |
| Modal pattern | `modalPattern` | string | YES | `flow` (Category 1 in MODAL-SPEC) |
| Primary CTA URL | `ctaUrl` | URL | YES | Gem or platform link |
| Primary CTA label | `ctaLabel` | string | YES | `LaForja` |
| Secondary CTA URL | `ctaUrl2` | URL | NO | Optional second tool |
| Secondary CTA label | `ctaLabel2` | string | NO | `LaReu` |

## 3. Flow Grouping Rules

Flows are organized into 4 groups that map to playbook sections.

### 3.1 Nucleo / Core (F1-F4) -- ALWAYS PRESENT

| Rule | Detail |
|---|---|
| Section ID | `nucleo` |
| Flows | F1, F2, F3, F4 |
| Display | `jarvis-card` set + optional `cycle-visual` |
| KA Levels | All KA-2 (entry level) |
| Frequency | Daily/weekly recurring |
| Table highlight | Default background (no special color) |
| Mandatory | YES -- every playbook must have a core group |

**Presentation pattern**:
- `jarvis-cards` grid (1 card per flow)
- Each card: `.jarvis-icon.{tool-class}` + flow number, h3 bilingual name,
  card body with tool/input/output/prompt/anti-use fields
- Optional `cycle-visual` showing the flow sequence visually

### 3.2 Investigacion / Investigation (F5-F6) -- OPTIONAL

| Rule | Detail |
|---|---|
| Section ID | `investigacion` |
| Flows | F5, F6 |
| Display | `jarvis-card` set |
| KA Levels | KA-3 (adaptation level) |
| Frequency | On-demand |
| Table highlight | `background:var(--sofka-orange-dim)` |
| Mandatory | NO -- include when playbook covers research-oriented flows |

**Omission criteria**: Brief explicitly marks `investigacion: false` AND no
research-type flows exist in the flow set.

### 3.3 Infraestructura / Infrastructure (F7-F10) -- ALWAYS PRESENT

| Rule | Detail |
|---|---|
| Section ID | `infra` |
| Flows | F7, F8, F9, F10 |
| Display | `jarvis-card` set + optional architecture box + `decision-table` |
| KA Levels | Mixed (KA-2 and KA-3) |
| Frequency | One-time setup + ongoing |
| Table highlight | `background:var(--sofka-info-dim)` |
| Mandatory | YES -- every playbook needs tool/platform setup flows |

**Presentation pattern**:
- Optional `vraid-box` or architecture callout at section start
- `jarvis-cards` grid (1 card per flow)
- Optional `decision-table` comparing tool configurations

### 3.4 Avanzado / Advanced (F11-F13) -- OPTIONAL

| Rule | Detail |
|---|---|
| Section ID | `avanzado` |
| Flows | F11, F12, F13 |
| Display | `jarvis-card` set + `gem-bar` |
| KA Levels | KA-3, KA-4, KA-5 (mastery levels) |
| Frequency | On-demand / continuous |
| Table highlight | `background:var(--sofka-positive-dim)` |
| Mandatory | NO -- include when playbook targets advanced practitioners |

**Omission criteria**: Brief explicitly marks `avanzado: false` AND no
mastery-level flows exist in the flow set.

## 4. Flow x Modal Mapping

Every flow N generates exactly one flow modal (`modal-f{N}`) following
MODAL-SPEC Category 1 (Flow Modal) structure. The modal MUST contain:

| Modal section | Source field |
|---|---|
| Purpose callout | `brief.problems[N]` + flow context |
| Timeline steps | 3-5 operational steps with time markers |
| Output description | `flow.outputEs` / `flow.outputEn` expanded |
| DoD callout | `flow.dod` from brief or kata DoD table |
| Week progression | Weeks 1-2, 4, 8 progression narrative |
| CTA links | `flow.ctaUrl` + `flow.ctaLabel` |

## 5. Flow Customization Rules for New Playbooks

When generating a playbook for a new topic (not Agile PM), the 13-flow
structure is a TEMPLATE. The following rules apply:

1. **Minimum 10 flows, maximum 16 flows** -- the 13 is a target, not a ceiling
2. **Core group (F1-F4 equivalent) is MANDATORY** -- at least 4 recurring operational flows
3. **Flow numbering is sequential** -- F1, F2, F3... no gaps
4. **Every flow must have a unique tool or tool combination** -- no two flows with identical tooling
5. **KA level distribution**: at least 4 flows at KA-2, at least 2 at KA-3, at least 1 at KA-4 or KA-5
6. **Frequency distribution**: at least 3 recurring (daily/weekly), at least 2 one-time, at least 2 on-demand
7. **Flow names must be action verbs** -- "Preparar", "Capitalizar", "Reportar" -- not nouns
