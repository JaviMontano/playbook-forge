# PLAYBOOK-SPEC v1.0 -- Entrusted Standard

> **Source of truth** for every HTML playbook generated by `playbook-forge`.
> Derived from the golden reference: `playbook_bu2_agile_pm_excellence_v3.html`
> (568 KB, 2820 lines, 69 modals, 12 sections, 1039 bilingual pairs).
>
> A playbook that fails ANY Gate 0-4 check is **BLOCKED** from delivery.
> The `playbook-reviewer` agent enforces this specification.
> If the assembler produces output that does not meet this spec, the assembler
> is wrong -- fix the assembler.

---

## 1. MANDATORY STRUCTURE

### 1.1 Document Shell

Every playbook is a **single self-contained HTML file**. No external CSS, no
external JS (fonts excepted).

| Requirement | Detail |
|---|---|
| DOCTYPE | `<!DOCTYPE html>` |
| html lang | `<html lang="es">` |
| body class | `class="lang-es"` |
| charset | `<meta charset="UTF-8">` |
| viewport | `<meta name="viewport" content="width=device-width, initial-scale=1.0">` |
| Google Fonts | `Inter` weights 300-900 via `fonts.googleapis.com` |
| Fontshare | `Clash Grotesk` weights 400-700 via `api.fontshare.com` |
| CSS | Complete **Sofka DS v5.1** inline `<style>` block containing ALL tokens (`:root` variables), ALL component classes, modal styles, bilingual toggle rules, interactive states, responsive breakpoints, and print styles |
| Skip link | `<a class="skip-link" href="#main">` for accessibility |

### 1.2 NAV (mandatory)

```
nav.toc
  .toc-inner
    .toc-logo          -- company name (e.g. "Sofka Technologies")
    a[href="#section"]  -- one per section (N links, N >= 10)
    button[onclick="toggleLang()"]  -- language toggle (aria-label required)
```

- Position: `sticky`, top: 0, z-index: 100
- Horizontal scroll with hidden scrollbar
- Nav link count MUST match section count

### 1.3 HERO (mandatory)

```
.hero  (background: black, border-bottom: 8px solid orange)
  .hero-inner
    .hero-logo         -- company name
    .hero-badges       -- min 3, max 7 badge spans
    h1 > span          -- orange highlight word(s), bilingual ES/EN
    .hero-sub          -- subtitle paragraph, bilingual
    .hero-kpis         -- grid of KPI cards
      .hero-kpi        -- min 4, max 6
        .hero-kpi-icon
        .hero-kpi-value
        .hero-kpi-label
```

**Golden reference counts**: 7 badges, 5 KPIs (grid-template-columns: repeat(5, 1fr)), 6 `.hero-kpi` elements.

### 1.4 SECTIONS (mandatory -- minimum 10, target 12)

Each section MUST have:
- `<section id="...">` with a stable anchor ID
- `.section-header` containing `h2` (bilingual, with `<span>` orange highlight) + `p` description (bilingual)
- `<hr class="section-divider">` after each section

#### 1.4.1 Mandatory Sections (every playbook)

| # | ID | Name | Key components |
|---|---|---|---|
| 1 | `glosario` | Glossary | `problem-grid` with 10+ clickable term cards opening modals + `gem-bar` |
| 2 | `agile-pm` (or context-specific profile ID) | Profile | `apm-equation` block showing value proposition |
| 3 | `flujos` | Flows | `map-table` with ALL use cases (13+ rows, each clickable to modal) |
| 4 | `katas` | Katas | `decision-table` (3-5 levels) + kata x flow activation matrix + callout cards + DoD table |
| 5 | `nucleo` | Core | `jarvis-card` set for primary use cases (F1-F4 equivalent) + optional cycle-visual |
| 6 | `antipatrones` | Anti-patterns | antipattern table with min 5, target 13 clickable rows opening modals |
| 7 | `ritmo` | Rhythm | `vraid-box` or equivalent framework box + 4 `metric-card` elements + timeline adoption + `acceptance-list` |
| 8 | `empezar` | Get Started | `problem-grid` with 5-6 empathy cards (4+ clickable to flow modals, 1 static tip) |
| 9 | `cierre` | Close | `decision-table` metadata (author, audience, team, version, pilot) + callout acknowledgments |

#### 1.4.2 Flow Group Sections (minimum 2 required)

At least 2 additional sections grouping flows by theme:
- Each contains `jarvis-card` elements for the flows in that group
- Example from golden reference: `nucleo` (F1-F4), `investigacion` (F5-F6), `infra` (F7-F10), `avanzado` (F11-F13)

#### 1.4.3 Optional Sections (add based on context)

| ID | Name | Use when |
|---|---|---|
| `investigacion` | Investigation | Research-oriented playbooks |
| `infra` | Infrastructure | Platform/tool setup playbooks |
| `avanzado` | Advanced | Mastery-level content |

**Golden reference section IDs** (12 total): `glosario`, `agile-pm`, `flujos`, `katas`, `nucleo`, `investigacion`, `infra`, `avanzado`, `antipatrones`, `ritmo`, `empezar`, `cierre`.

### 1.5 MODALS (mandatory -- minimum 30, target 50+)

**Golden reference**: 69 unique modals.

Every modal follows this shell:

```html
<div class="modal-overlay" id="modal-{prefix}{id}">
  <div class="modal-box">
    <button class="modal-close" onclick="closeModal()" aria-label="Cerrar / Close">&times;</button>
    <div class="modal-header">
      <h3><span class="es">...</span><span class="en">...</span></h3>
      <small><span class="es">...</span><span class="en">...</span></small>
    </div>
    <div class="modal-body">
      <!-- type-specific content -->
    </div>
  </div>
</div>
```

#### 1.5.1 Flow Modal (`id="modal-f{N}"`)

Internal structure (10 sections, ALL required):

1. `callout-warning`: "Para que lo usas?" / "What is it for?"
2. `h4` bilingual: "Como funciona" / "How it works"
3. `timeline`: 3-5 steps with time markers (0-5 min, 5-15 min, etc.)
4. `h4` bilingual: "Que obtienes al final" / "What you get at the end"
5. `p` bilingual: output description
6. `callout callout-orange` (or callout-info): "Como sabes que lo hiciste bien" / "How you know you did it right"
7. `h4` bilingual: "Cuando vas a notar la diferencia" / "When you'll notice the difference"
8. `p` bilingual: Week 1-2 / Week 4 / Week 8 progression
9. (optional) additional callouts for edge cases
10. `modal-cta`: `gem-link` buttons (primary with `.pulse` class)

**Golden reference**: 13 flow modals (`modal-f1` through `modal-f13`).

#### 1.5.2 Anti-pattern Modal (`id="modal-fap{N}"`)

Internal structure (5 sections, ALL required):

1. `callout-warning`: "Que es" / "What it is"
2. "Por que pasa" / "Why it happens"
3. "Como detectarlo" / "How to detect it"
4. `timeline`: 3 remediation steps
5. `callout-critical`: accountability statement

**Golden reference**: 13 anti-pattern modals (`modal-fap1` through `modal-fap13`).

#### 1.5.3 Glossary Modal (`id="modal-fg-{slug}"`)

Internal structure:

1. `callout-info`: "Concepto" / "Concept"
2. `callout` orange variant: "Para que te sirve" / "Why it matters"
3. "Ejemplo practico" / "Practical example" (optional timeline)

**Golden reference**: 16 glossary modals.

#### 1.5.4 Kata Modal (`id="modal-fka{N}"`)

Internal structure:

1. `callout-warning`: "Que significa" / "What it means"
2. `timeline`: 4 trial progression (Trial 1-4)
3. `callout-orange`: "Criterio de superacion" / "Advancement criterion"

**Golden reference**: 5 kata modals (`modal-fka1` through `modal-fka5`).

#### 1.5.5 Other Modal Types (context-dependent)

| Prefix | Purpose | Golden ref count |
|---|---|---|
| `modal-fdm{N}` | Decision-matrix entries | 7 |
| `modal-fl-{slug}` | Tool/link modals | 3 |
| `modal-fm-{slug}` | Metric deep-dive modals | 7 |
| `modal-frv-{slug}` | Rhythm/vision modals | 3 |
| `modal-fimpacto` | Impact summary modal | 1 |

### 1.6 FOOTER (mandatory)

```
footer.site-footer  (background: black, border-top: 8px solid orange)
  .footer-inner
    .footer-logo      -- company name
    .footer-badges    -- 3 badge spans
    .footer-bottom    -- copyright + author (bilingual)
```

### 1.7 JS RUNTIME (mandatory)

Single `<script>` block at end of `<body>`, wrapped in IIFE:

```javascript
(function(){
  // 1. Restore language from localStorage (key: 'wr-lang')
  // 2. toggleLang() -- swaps lang-es/lang-en on body, persists to localStorage
  // 3. openModal(id) -- resolves ALL prefixes:
  //      numeric     -> modal-f{id}
  //      ap{N}       -> modal-fap{N}
  //      g-{slug}    -> modal-fg-{slug}
  //      ka{N}       -> modal-fka{N}
  //      dm{N}       -> modal-fdm{N}
  //      l-{slug}    -> modal-fl-{slug}
  //      m-{slug}    -> modal-fm-{slug}
  //      rv-{slug}   -> modal-frv-{slug}
  //      catch-all   -> modal-f{id}
  // 4. closeModal() -- removes .active from all overlays, restores body overflow
  // 5. Event: click on .modal-overlay -> closeModal()
  // 6. Event: Escape key -> closeModal()
  // 7. Event: Enter/Space on role="button" -> e.target.click()
  // 8. Expose: window.toggleLang, window.openModal, window.closeModal
})();
```

**Critical**: The `openModal` function uses `document.getElementById('modal-f' + id)` as the base resolution. All modal IDs MUST follow the `modal-f{suffix}` pattern where the suffix is what gets passed to `openModal()`.

### 1.8 ACCESSIBILITY (mandatory)

| Requirement | Detail |
|---|---|
| `role="button"` | On ALL clickable non-anchor elements (cards, table rows) |
| `tabindex="0"` | On ALL `role="button"` elements |
| `aria-label` | On all modal close buttons (`"Cerrar / Close"`) |
| `aria-label` | On language toggle button |
| `:focus-visible` | Outline rule in CSS (`2px solid var(--sofka-orange)`) |
| Skip link | `<a class="skip-link" href="#main">` |
| Semantic HTML | `<section>`, `<nav>`, `<main>`, `<header>` (implicit in hero), `<footer>` |

**Golden reference**: 137 `role="button"` elements, 70 `aria-label` attributes.

### 1.9 BILINGUAL (mandatory)

Every user-visible text element uses paired spans:

```html
<span class="es">Texto en espanol</span><span class="en">English text</span>
```

CSS rules:
```css
body.lang-es .en { display: none !important; }
body.lang-en .es { display: none !important; }
```

- Default language: `lang-es` (body class `lang-es`)
- Toggle persists to `localStorage` key `wr-lang`
- ES/EN span counts must be balanced (within +/- 5)

**Golden reference**: 1039 `.es` spans, 1040 `.en` spans.

## 3. NAMING CONVENTIONS

### 3.1 File Naming

```
playbook_{unit}_{slug}_v{version}.html
```

Examples:
- `playbook_bu2_agile_pm_excellence_v3.html`
- `playbook_bu2_pb3-lavuelta-capitalizar-decisiones_v2.html`

### 3.2 Section IDs

Lowercase, no accents, hyphenated: `glosario`, `agile-pm`, `flujos`, `katas`,
`nucleo`, `investigacion`, `infra`, `avanzado`, `antipatrones`, `ritmo`,
`empezar`, `cierre`.

**NOTE**: The golden reference has accent marks in some IDs (`investigacion`,
`antipatrones`). New playbooks SHOULD use ASCII-only IDs for maximum
compatibility.

### 3.3 Modal IDs

All modal IDs follow the pattern `modal-f{suffix}`:

| Type | ID pattern | openModal() call |
|---|---|---|
| Flow | `modal-f{N}` | `openModal('{N}')` |
| Anti-pattern | `modal-fap{N}` | `openModal('ap{N}')` |
| Glossary | `modal-fg-{slug}` | `openModal('g-{slug}')` |
| Kata | `modal-fka{N}` | `openModal('ka{N}')` |
| Decision-matrix | `modal-fdm{N}` | `openModal('dm{N}')` |
| Tool/link | `modal-fl-{slug}` | `openModal('l-{slug}')` |
| Metric | `modal-fm-{slug}` | `openModal('m-{slug}')` |
| Rhythm/vision | `modal-frv-{slug}` | `openModal('rv-{slug}')` |
| Impact | `modal-fimpacto` | `openModal('impacto')` |

### 3.4 CSS Class Conventions

All classes come from **Sofka DS v5.1**. No custom class additions. If a new
visual pattern is needed, compose from existing DS classes.

## 4. QUALITY GATES (Verification Checklist)

### Gate 0: Structure (automated)

- [ ] **G0.1** -- 10+ `<section` tags with `id` attributes
- [ ] **G0.2** -- 30+ `modal-overlay` elements
- [ ] **G0.3** -- `body` has `class="lang-es"` (or class containing `lang-es`)
- [ ] **G0.4** -- JS `<script>` block present with `toggleLang`, `openModal`, `closeModal`
- [ ] **G0.5** -- 0 unreplaced `{{PLACEHOLDER}}` or `<!-- SWARM:` markers
- [ ] **G0.6** -- File size > 150 KB

### Gate 1: Content Density (automated)

- [ ] **G1.1** -- 40+ `timeline-item` elements
- [ ] **G1.2** -- 30+ `callout` elements
- [ ] **G1.3** -- 15+ `problem-card` elements
- [ ] **G1.4** -- 8+ `jarvis-card` elements
- [ ] **G1.5** -- 20+ `clickable-card` elements
- [ ] **G1.6** -- 5+ `gem-link` elements

### Gate 2: Bilingual Completeness (automated)

- [ ] **G2.1** -- `span.es` count == `span.en` count (+/- 5)
- [ ] **G2.2** -- All `h2` elements contain bilingual pairs (both `.es` and `.en` spans)
- [ ] **G2.3** -- All modal-header `h3` elements contain bilingual pairs
- [ ] **G2.4** -- Language toggle button present in nav

### Gate 3: Accessibility (automated)

- [ ] **G3.1** -- 20+ `role="button"` elements
- [ ] **G3.2** -- All `modal-close` buttons have `aria-label`
- [ ] **G3.3** -- Skip link present (`class="skip-link"`)
- [ ] **G3.4** -- `:focus-visible` rule in CSS

### Gate 4: Mandatory Sections (automated)

- [ ] **G4.1** -- `section#glosario` present
- [ ] **G4.2** -- `section` with kata-like content present (id contains `kata`)
- [ ] **G4.3** -- `section` with antipattern content present + 5+ clickable rows
- [ ] **G4.4** -- `section` with rhythm/adoption content present (id contains `ritmo` or `rhythm`)
- [ ] **G4.5** -- `section#empezar` present with 4+ card elements inside
- [ ] **G4.6** -- `section#cierre` present with metadata table

### Gate 5: Modal Quality (manual spot-check, recommended)

- [ ] **G5.1** -- Sample flow modal has all 10 internal sections (purpose, how-it-works timeline, output, DoD callout, progression, CTA)
- [ ] **G5.2** -- Sample anti-pattern modal has all 5 sections (concept, why, detect, remediation timeline, accountability callout)
- [ ] **G5.3** -- All sampled modals have bilingual content (both `.es` and `.en` spans)

### Gate 6: Size Target (soft target)

- [ ] **G6.1** -- File size between 150 KB and 600 KB
- [ ] **G6.2** -- 200+ unique bilingual pairs (`span.es` count >= 200)
